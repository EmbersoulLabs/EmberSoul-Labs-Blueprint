# PD-023 — Workflow Engine Architecture

## Status

Locked

## Decision

EmberOS V1 backend must be structured as a Workflow Engine.

## Architecture

Frontend → Workflow Engine → AI Skills → Prompt Library → AI Provider

## Rules

- Frontend must not contain prompt logic.
- Workflow Engine decides execution order, retry, failure handling, self challenge, and output flow.
- AI Skills must remain modular.
- Prompt Library must be version controlled.

## Reason

This prevents EmberOS from becoming a simple GPT wrapper and keeps the system maintainable.
