# PD-022 — EmberOS Is Not a Chat Application

## Status

Locked

## Decision

EmberOS V1 will not expose a ChatGPT-style chat interface to end users.

User interaction should be workflow-based:

```text
Upload Video
  ↓
Campaign Information
  ↓
Generate
  ↓
Marketing Report
  ↓
Export / Schedule / Regenerate
```

## Reason

SME users do not want to chat with AI or learn prompt engineering.

They want EmberOS to behave like an automatic AI Marketing Team.

## Product Implication

- Frontend should avoid chat-window-first interaction.
- Prompt engineering should stay inside the backend and Prompt Library.
- User-facing flow should remain simple, guided, and button-based.
