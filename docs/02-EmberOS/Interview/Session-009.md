# Session 009

## Topic

Workflow Engine, Product Specification, and Prompt Engineering workflow.

## User Decisions

1. EmberOS app should not use ChatGPT-style chat windows.
2. User experience should be button-based and workflow-driven.
3. Users upload video, fill campaign information, and click Generate.
4. AI Skills run automatically in the backend.
5. ChatGPT is used for development, prompt design, blueprint maintenance, and review, not as the customer-facing UI.
6. Product development should stop expanding features and move into Product Specification.
7. Development order is:
   - Schema
   - Workflow
   - AI Skills
   - Prompt
   - API
   - Frontend
   - Testing
   - Release
8. Prompt Engineers should maintain one prompt per file.
9. Prompt work must not create new product decisions.
10. Blueprint and Product Decision are the Single Source of Truth.

## Notes

This session locks the shift from broad product discussion into implementation-ready specification work.

The focus is now maintainability, version control, and clear execution for AI Skills, Workflow Engine, Prompt Library, API, frontend, testing, and release.
